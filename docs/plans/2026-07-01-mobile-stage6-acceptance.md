# 2026-07-01 mobile-minutes 阶段 6 验收计划

## 目标

把手机端 `/mobile-minutes` 从“页面和接口已接上”推进到“可以按真实业务路径验收”。

## 验收原则

- 先只读验收，再写入验收。
- 写入验收只使用明确的测试会议 / 测试待办，避免污染正式业务。
- 手机端消息必须对齐后台消息通知的接收口径，不直接展示原始 `activityLogs`。
- 每轮修复后必须通过 `corepack pnpm run build` 和公网 `/mobile-minutes` 检查。

## 当前已发现并处理

1. 移动端消息原先直接使用 `activityLogs`。
   - 风险：员工账号会看到不是发给自己的业务日志，和后台“消息通知”不一致。
   - 处理：改为从会议、待办和活动记录生成后台通知口径的手机端消息，并按当前登录用户过滤。
