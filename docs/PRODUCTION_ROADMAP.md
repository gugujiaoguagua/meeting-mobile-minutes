# 从 Demo 到正式系统的开发路线

本文档按阶段拆分正式版建设工作，方便技术团队排期。

## 阶段 0：Demo 交接与需求冻结

目标：把当前 Demo 转成团队可开发的需求基线。

建议产出：

- 页面清单。
- 状态流转图。
- 数据字段表。
- 权限矩阵。
- AI 输出 JSON 样例。
- 企业微信通知样例。
- 第一阶段不做事项清单。

预计时间：2-3 天。

## 阶段 1：后端与数据模型

目标：让会议、待办、审批、消息可以真实保存。

优先数据表：

- users：用户。
- departments：部门。
- meetings：会议。
- meeting_files：会议上传文件。
- meeting_minutes：会议纪要。
- meeting_decisions：会议决策。
- tasks：统一待办。
- task_approval_logs：待办签批日志。
- task_review_logs：待办复核日志。
- notifications：消息通知。
- dictionaries：会议词典。
- okr_projects：OKR 项目。
- okr_krs：KR。
- okr_pdca_tasks：PDCA 任务。

预计时间：1-2 周。

## 阶段 2：会议闭环 MVP

目标：完成会议上传到待办闭环的最小生产版本。

必须完成：

- 会议新建。
- 文件上传。
- AI 纪要生成。
- AI 待办生成。
- 主管修正。
- 管理者逐条签批。
- 正式待办台账。
- 推进人提交完成。
- 复核人确认完成。
- 消息通知。
- 管理驾驶舱基础统计。

预计时间：2-3 周。

## 阶段 3：企业微信接入

目标：让员工不用重复登录和重复查看系统。

建议顺序：

1. 企业微信扫码或 SSO 登录。
2. 同步组织架构。
3. 同步用户、部门、主管关系。
4. 站内消息对接企业微信机器人。
5. 待办提醒推送到个人或群。

预计时间：1-2 周。

## 阶段 4：OKR 项目闭环

目标：把 OKR 项目里的 KR 和 PDCA 纳入统一待办系统。

必须完成：

- 创建 OKR 项目。
- 创建 KR。
- 创建 PDCA 任务。
- PDCA 任务进入推进人的我的待办。
- PDCA 任务完成后进入复核人待办。
- KR 下 PDCA 全部完成后，推送给 OKR 负责人复核。
- KR 复核通过后纳入 OKR 进度统计。

预计时间：1-2 周。

## 阶段 5：看板与数据治理

目标：让管理驾驶舱具备真实经营价值。

必须统一口径：

- 会议召开数量。
- 会议上传率。
- 会议时长。
- 会议人工时。
- 任务新增数。
- 已完成数。
- 已提交待复核数。
- 逾期数。
- 公司支持事项数。
- 部门维度。
- 门店维度。
- OKR 维度。

建议增加：

- 日期自定义筛选。
- 部门筛选。
- 人员筛选。
- 逾期穿透。
- 公司支持穿透。
- 导出报表。

预计时间：1-2 周。

## 阶段 6：生产稳定性

目标：系统可以真实长期运行。

必须补齐：

- 操作日志。
- 审批日志。
- 错误重试。
- 文件解析失败处理。
- AI 生成失败处理。
- 权限越权拦截。
- 数据备份。
- 性能优化。
- 监控告警。

预计时间：2-4 周。

## 第一版建议不做

为保证项目快速上线，以下内容建议后置：

- 真实语音识别。
- 多模型复杂路由。
- 飞书完整兼容。
- 移动端完整适配。
- 复杂 BI 报表。
- 精细到字段级的权限。
- 自动判断会议质量。

## 技术建议

前端：

- 保留 Next.js + TypeScript + Tailwind CSS。
- 当前 Demo 可作为页面和交互参考，但正式版建议拆组件、拆状态、接 API。

后端：

- 推荐使用 PostgreSQL 或 MySQL。
- 待办、会议、OKR 都应由后端统一计算状态。

AI：

- 第一阶段固定输出 JSON。
- AI 结果必须允许主管修正。
- AI 不应直接写入正式台账，必须经过签批。

权限：

- 先做角色权限和数据范围权限。
- 管理者可看全部或授权范围。
- 部门主管看本部门。
- 员工看自己相关任务和会议。

消息：

- 站内消息先行。
- 企业微信机器人后接。
- 所有关键状态变化都要生成消息记录。

## 验收标准

第一阶段上线前，至少要能完成以下真实闭环：

1. 上传一场会议文稿。
2. AI 生成纪要和待办。
3. 主管修正待办。
4. 管理者签批通过。
5. 待办进入正式台账。
6. 推进人在我的待办中提交完成。
7. 复核人在我的待办中确认完成。
8. 管理驾驶舱数据同步变化。
9. 消息通知产生对应提醒。
10. 企业微信收到至少一种状态变更提醒。
